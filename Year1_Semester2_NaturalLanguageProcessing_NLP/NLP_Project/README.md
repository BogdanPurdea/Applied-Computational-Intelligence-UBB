# Transformer NLP Classification Project

This project implements three distinct transformer-based classification models using PyTorch and HuggingFace Transformers.

## Structure

```
nlp_transformer_classification/
│
├── 1_BERT_IMDB/
│   └── main.py                        # BERT for binary sentiment analysis
├── 2_SENTENCEBERT_FAKE/
│   └── main.py                        # Sentence-BERT + MLP for fake news
├── 3_ROBERTA_RED/
│   └── main.py                        # RoBERTa for multi-class emotion classification
├── utils.py                           # Shared metric, LIME, SHAP and data logic
├── requirements.txt
├── README.md
│
├── data/                              # Placeholder for datasets
│   ├── imdb/
│   ├── welfake/
│   └── romanian_emotion/
│
└── outputs/                           # Autogenerated model outputs and metrics
```

## Setup & Requirements

### Hardware Requirements
- **GPU Recommended**: Training transformer models on CPU is exceedingly slow. A CUDA-compatible GPU is highly recommended.
- Memory: Minimum 8GB VRAM is advised for BERT and RoBERTa fine-tuning.

### Installation
Install the necessary dependencies:

You need to download the datasets manually and provide their paths via CLI arguments.

1. **IMDB Movie Reviews**: Download from Kaggle. Expected to be a CSV file with `review` and `sentiment` columns.
2. **WELFake Dataset**: Download from Kaggle or the original source. Expected to have `title`, `text`, and `label` columns.
3. **Romanian Emotion Dataset**: Expected to be a CSV file containing `text` and `emotion` columns.

## Running the Models

Each script provides configurable CLI arguments. You can pass `--help` to see all available options.

### 1. IMDB Sentiment (BERT)
```bash
python 1_BERT_IMDB/main.py \
    --epochs 3 \
    --batch_size 16 \
    --run_lime true \
    --run_shap true
```

### 2. WELFake Fake News (Sentence-BERT + MLP)
```bash
python 2_SENTENCEBERT_FAKE/main.py \
    --epochs 5 \
    --learning_rate 1e-3 \
    --freeze_encoder \
    --run_lime true
```
*(Omit `--freeze_encoder` to fine-tune the Sentence-BERT backbone along with the MLP)*

### 3. Romanian Emotion Classification (RoBERTa)
```bash
python 3_ROBERTA_RED/main.py \
    --epochs 3 \
    --run_lime true
```

## LIME and SHAP Explainability

Interpretability methods can be enabled using `--run_lime true` and `--run_shap true`. 
- **LIME** output consists of HTML files showing the word importance for the prediction.
- **SHAP** outputs visualization plots showing feature attributions.

> **Note:** Generating SHAP/LIME explanations for transformer models can be very compute-intensive. Use the `--num_explainer_samples` argument (default is 5) to restrict the number of samples explained.

## Artifacts Generated

After running, check the respective `outputs/` folder (e.g. `outputs/bert_imdb/`) for:
- `best_model.pt` or `pytorch_model.bin`
- Tokenizer files
- `test_metrics.json`
- `test_confusion_matrix.png`
- `classification_report.txt` (Multi-class)
- `lime_explanations/` folder
- `shap_explanations/` folder

## Known Limitations
- Sentence-BERT classification currently loads all tokenized data sequentially. Freezing the encoder yields faster results, but unfreezing it requires significantly more VRAM.
- LIME and SHAP explainers run sequentially and may take several minutes per sample on CPU.
